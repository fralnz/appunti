Asincronicità in Javascript
...

In Javascript, il linguaggio è strutturato intorno agli eventi.
Per gestire gli eventi, si possono definire delle funzioni callback che vengono eseguite in risposta all'evento.
Le callback non vengono eseguite immediatamente ma vengono memorizzate in un registro interno e vengono eseguite solo quando l'evento si verifica e viene concluso.

Esecuzione di un programma in Javascript
...

Javascript utilizza una pila per eseguire le varie istruzioni, seguendo il principio LIFO (Last In, First Out).
Le funzioni vengono inserite nella pila di esecuzione e vengono eseguite in ordine di inserimento.
Ogni funzione rimane nella pila finché tutte le sue istruzioni non sono state eseguite.

Callback
...

Le callback sono funzioni che vengono richiamate da altre funzioni, spesso in risposta a un evento o al completamento di un task.
Le callback sono utilizzate per gestire l'asincronicità in Javascript.
Quando un'operazione asincrona viene completata, la callback associata viene eseguita.
Esempio:

function greeting(name, callback) {
	console.log('Ciao, ' + name + '!');
	callback();
}

function sayGoodbye() {
	console.log('Arrivederci!');
}

Web API
...

Javascript, essendo un linguaggio single-threaded, utilizza le Web API del browser per eseguire operazioni asincrone.
Le Web API consentono di eseguire operazioni come ricevere dati da un server o inizializzare un timer su un thread separato dal thread principale di esecuzione di Javascript.

Callback hell
...

Callback hell è un problema che si verifica quando si annidano troppe callback una dentro l'altra, rendendo il codice difficile da leggere e da gestire.
Questo problema può essere risolto scrivendo le callback in funzioni separate e assegnando loro nomi descrittivi.

Ajax
...

AJAX (Asynchronous JavaScript and XML) è un insieme di strumenti che permettono di lavorare in modo asincrono, in particolare per interagire con server esterni tramite chiamate HTTP.
L'oggetto XMLHttpRequest è stato il primo strumento utilizzato per eseguire richieste asincrone, consentendo agli sviluppatori di aggiornare parti specifiche di una pagina web senza dover ricaricare l'intera pagina.

Promise
...

Le promise sono un modo per gestire le operazioni asincrone in modo più chiaro rispetto alle callback.
Una promise è un oggetto che rappresenta il risultato di un'operazione asincrona.
Le promise forniscono metodi per gestire il successo o il fallimento dell'operazione asincrona, come ad esempio then() per il successo e catch() per il fallimento.
Esempio:

const fetchData = () => {
    return new Promise(resolve => {
        setTimeout(() => {
            resolve('Dati ricevuti!');
        }, 2000);
    });
};

fetchData().then(data => console.log(data));

async - await
...

async e await sono keywords introdotte in Javascript per gestire le operazioni asincrone in modo più leggibile e simile al codice sincrono.
async viene utilizzato per definire una funzione asincrona, mentre await viene utilizzato all'interno di una funzione asincrona per attendere il risultato di una promise senza dover utilizzare then().
Anche utilizzando async e await, si stanno sfruttando le promise sottostanti.
Esempio:

const fetchDataAsync = async () => {
    const data = await fetchData();
    console.log(data);
};

fetchDataAsync();

I form
...

I form sono il meccanismo principale di interazione utente-sito nel web, consentendo agli utenti di inviare informazioni al server.
I form permettono di creare una struttura di elementi in cui gli utenti possono inserire i loro dati.
Questi elementi mettono in relazione il sito web con l'utente, consentendo lo scambio di informazioni utili ad entrambi.

<form>
    <label>Username: <input type="text"></label><br>
    <label>Password: <input type="password"></label><br>
    <input type="submit" value="Invia">
</form>;

Elementi HTML per i form
...

Gli elementi HTML per i form includono una serie di input field che permettono agli utenti di inserire diversi tipi di dati.
Tra i tipi di input più comuni ci sono text, checkbox, select, radio button, ecc.
Ogni tipo di input ha un comportamento specifico e può essere personalizzato attraverso gli attributi HTML.

Tipi di input HTML
...

I tipi di input HTML includono una vasta gamma di opzioni come text, button, checkbox, date, email, file, hidden, image, password.
