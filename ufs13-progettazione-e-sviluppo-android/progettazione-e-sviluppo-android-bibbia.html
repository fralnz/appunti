Componenti di Android
...

  • Activity: sono delle schermate della nostra applicazione, (in iOS View), definiamo un controllore con l'obbiettivo di gestire I/O di una determinata applicazione.
  • Service: sono delle classi controllare che mi consentono di utilizzare le applicazioni in background, ma vengono eseguite in un processo separato. Sono dei componenti che non fanno uso dell'interfaccia grafica, ci deve essere una comunicazione.
  • Broadcast receiver: consente di registrare la mia applicazione e di ricevere segnali rivolti a tutte le app in esecuzione, anche se non utilizza l'interfaccia del sistema possono far apparire messaggi informativi che si sovrappongono all'output dell'activity corrente.
  • Content provider: un componente che mi consente di accedere alle informazioni o del sistema operativo o in altre applicazioni,  dati possono essere contenuti nel file system, in un database SQLite del web o in una qualunque locazione di dati
    Application Framework → ciclo infinito, cona fila di messaggi che devono essere ricevuti.

Activity
...

Definizione

Activity: Elementi che cooperano tra loro o che possono essere indipendenti l'una delle altre.

Ciclo di vita
...

dipende dal sistema operativo o di altre applicazioni

  1. Nasce
  2. Cresce
  3. Muore
    Ma essa può essere messa in pausa (quando la schermata è parzialmente invisibile), ma viene stoppata quando cambio schermata dell'applicazione oppure proprio applicazione. Ma posso evocare lo stop e farlo riprodurre in background.

Codice
...

Possiamo implementare la nostra Activity:

public abstract class Activity extends ApplicationContext{...}

Stati
...

Le activity sono imprevedibili: non sappiamo quando possono crashare. Pertanto Android obbliga le app a fare salvataggi di stato periodici.

Intent
...

Definizione

Intent: comunicazione al sistema operativo di un intento di passare da una schermata alla successiva.

Le schermate sono dichiarate nel manifest.
Nel manifest si dichiara anche l'intento delle activity.

Intent intent = new Intent(this, MiaActivity.class);
startActivity(intent);


Intent esplicito: specifica i componenti che devono essere avviati → comunica al SO di voler creare un servizio.
Intent implicito: esprimo solo l'azione che voglio fare.

Build
...


Gli apk sono degli zip rinominati. All'interno di un apk sono contenuti:

  • AndroidManifest.xml: contiene tutti i riferimenti di schermate disponibili con permessi annessi.
  • resources.arsc: file della grafica compilata. Contiene file come immagini, font, video, audio...
  • file .dex: bytecode cifrato. Le nostre applicazioni devono essere sempre firmate, anche in modalità debug.
  • risorse decompilate.
    Una volta decompressi, gli apk vengono installati con ADB (Android Device Build).

Definizioni e considerazioni
...

  • aapt: Android Asset Packaging Tool: Legge gli xml e genera R.java
  • aidl: Android Interfce Definition Language, converte interfacce di servizi .aidl in interfacce Java
  • Tutto il codice Java è compilato generando bytecode.class.
  • dex: converte i .class in file .dex eseguibili da Dalvik/ART
  • apkbuilder: comprime e impacchetta i .dex e le risorse in un unico file .apk.
  • jardesigner: permette di inserire una firma nell'apk.
  • zipalign: consente di ottimizzare le risorse di memoria utilizzata dall'applicazione di un dispositivo.

Firma
...

Ogni applicazione Android ha bisogno di una firma privata che identifichi l'autore per essere pubblicata, generando quindi fiducia.

Nota

Non rilasciare mai le applicazioni in debug mode! Si può fare il reverse engineering sull'apk ed ottenere i file sorgenti.

Relative Layout
...

Il RelativeLayout permette di usare posizioni indipendenti dalla dimensioni dello schermo.

<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="horizontal"
    android:weightSum="10">
    <View
        android:layout_width="0dp"
        android:layout_height="50dp"
        android:layout_weight="3"/>
    <View
        android:layout_width="0dp"
        android:layout_height="50dp"
        android:layout_weight="5"/>
    <View
        android:layout_width="0dp"
        android:layout_height="50dp"
        android:layout_weight="2"/>
</LinearLayout>

weightSum esprime il "peso" totale del contenitore e layout_weight esprime parte del peso che copre quel componente.

Nota

Mettiamo 0dp al valore che venga espresso in percentuale (in questo caso weight)


L'unità di misura utilizzata sono i dp (Density Independent Pixel).

Gestione dell'orientamento dello schermo
...

Ogni volta che si cambia orientamento del telefono, viene cambiata e ricaricata la View.
Chiamiamo il metodo onConfigurationChanged ogni volta che viene cambiato lo schermo. Per essere notificati del fatto che si cambia la forma della schermata, si va a scrivere nel manifest:

<activity
android:name=".NomeActivity"
android:label="@string/app_name"
andorid:configChanges="orientation|keyboardHidden">

Per forzare l'orientamento della schermata:

//Portrait (Verticale)
setRequestedOrientation(ActivityInfo.SCREEN_ORIENTATION_PORTRAIT)
//Landscape (Orizzontale)
setRequestedOrientation(ActivityInfo.SCREEN_ORIENTATION_LANDSCAPE)

Per nascondere la tastiera:

getWindow().setInputMode(WindowManager.LayoutParams.SOFT_INPUT_STATE_ALWAYS_HIDDEN)

Per salvare le informazioni prima che venga cambiata la schermata, si usa onSaveInstanceState con collezioni di coppie chiave-valore.
Se vogliamo salvare dati specifici non previsti da onSaveInstanceState, si fa l'override di quest'ultima.

Definizione

Sandbox: isolamento per il quale i file dell'applicazione devono essere accessibili solo e soltanto dall'applicazione stessa.

Memoria interna
...

Per salvare in maniera persistente le scelte dell'utente si usano le Shared Preference.
Le altre metodologie di memorizzazione persistente dei dati sono:

  • Internal Storage: salva i dati con accesso privato nella memoria del device.
  • External Storage: permette di salvare i dati con accesso public in una memoria esterna condivisa (scheda SD).
  • Network Connection: salva i dati sul web.
  • Database SQLite: salva i dati in un DB.
    All'interno della sandbox un'applicazione ha una quota limitata di spazio da utilizzare (una volta era da 50 ai 200 MB, ora è di più).

Classe Context
...

La classe android.content.Context contiene metodi di accesso e lettura file della sandbox.

Lettura File
...

Alcuni metodi:

  • getFilesDir(): restituisce tutti i file all'interno di una directory.
  • getAbsolutePath(): ottiene il percorso assoluto di una applicazione.
    ES: /data/data/<package applicazione>/files.

Scrittura File
...

Per scrivere all'interno dell'area:

public FileOutputStream openFileOutput (String name, int mode) throws FileNotFoundException

Il parametro name è il nome del file da scrivere, mentre il parametro mode può essere:

  • Context.MODE_PRIVATE: rende il file privato, cioè utilizzabile solo dall'applicazione.
  • Context.MODE_APPEND: agisce in append sul file (accoda dati).
  • `Context.MODE_WORLD_READABLE
  • Context.MODE_WORLD_WRITEABLE Oltre adopenFileOutputesiste il metodoopenFileInput`:
public FileInputStream openFileInput (String name, int mode) throws FileNotFoundException

Per eliminare i file:

public boolean deleteFile(String name)

Per elencare i file all'interno della directory dell'applicazoine:

public String[] fileList()

Storage esterno
...

Dato che non sappiamo se l'external storage esiste o no, possiamo verificarlo con:

public static String getExternalStorageState()

La stringa restituita può essere confrontata con una delle seguenti costanti:

  • Enviroment.MEDIA_MOUNTED: Lo storage esterno è disponible e pronto
  • Enviroment.MEDIA_UNMOUNTED: Lo storage esterno non è disponibile.
    Una volta che si è accertati che lo storage esterno esista:
public static File getExternalStorageDirectory()

Shared Preference
...

Un file xml che viene salvato all'interno di /data/data/<package>.

public abstract SharedPreferences getSharedPreferences(String name, int mode)

Se non vogliamo specificare il nome si può usare il metodo:

public SharedPrefences getPreferences(int mode)

In questo caso il nome sarà lo stesso dell'activity.
Per salvare le preferences si usa il metodo commit().

SQLite
...

SQLite è un database leggerissimo a singolo file.
Il file del DB viene salvato in data/data/packagename/databases.
SQLite fa uso del concetto transazionale.
La classe ContactDbAdapter fornisce l'interfaccia per un adapter al DB.
Per fare le query si usa la funzione query(), la quale restituisce un oggetto di tipo Cursor.
Un Adapter gestisce il modello dati e li adatta nelle righe individuali della collezioni di dati della finestra.
Ogni linea nella list view consiste di un layout che può essere complesso quanto si vuole.

Definizione

Adapter: un pattern, consente di associare il dato all'interno di una collezione (come ArrayList) ad un template grafico. Viene ripetuto tante volte quanti sono i dati tra il widget del template con il valore dell'attributo del nostro dato.

Cursor Adapter
...

Per visualizzare oggetti complessi ricevuti dal database si utilizzano i CursorAdapter, che vengono utilizzati nelle ListView e altri componenti.

import android.content.Context;
import android.database.Cursor;
import android.view.LayoutInflater;
import android.view.View;
import android.view.ViewGroup;
import android.widget.CursorAdapter;
import android.widget.TextView;
public class CustomCursorAdapter extends CursorAdapter {
  public CustomCursorAdapter(Context context, Cursor c) {
    super(context, c);
  }
  @Override
  public View newView(Context context, Cursor cursor, ViewGroup parent) {
    // when the view will be created for first time,
    // we need to tell the adapters, how each item will look
    LayoutInflater inflater = LayoutInflater.from(parent.getContext());
    View retView = inflater.inflate(R.layout.single_row_item, parent, false);
    return retView;
  }
  @Override
  public void bindView(View view, Context context, Cursor cursor) {
    // here we are setting our data
    // that means, take the data from the cursor and put it in views
    TextView textViewPersonName =
        (TextView) view.findViewById(R.id.tv_person_name);
    textViewPersonName.setText(
        cursor.getString(cursor.getColumnIndex(cursor.getColumnName(1))));
    TextView textViewPersonSurname =
        (TextView) view.findViewById(R.id.tv_person_surname);
    textViewPersonSurname(
        cursor.getString(cursor.getColumnIndex(cursor.getColumnName(2))));
  }
}

Gestione dei thread
...

Per gestire i thread secondari si utilizza la classe AsyncTask, che mette a disposizione un'interfaccia che include:

  • doInBackground: fa eseguire istruzioni in un thread secondario.
  • onPreExecute: inizializza le operazioni prima che avvenga l'esecuzione di doInBackground. Spesso si prepara la ProgressDialog e viene mostrata.
  • onPostExecute: viene eseguito alla fine di doInBackground ed anch'esso svolge operazioni collegate all'interfaccia utente.
  • onProgressUpdate: viene invocato ogni volta che dall'interno di doInBackground viene chiamato publicProgress (che costringe a comunicare con il thread principale). Serve a fornire aggiornamenti periodici all'interno dell'interfaccia utente ed in questo caso a spostare la barra di progresso.
    Questi ultimi tre vengono eseguiti nel thread principale e si occupano di comunicazione tra thread.

Esempio
...

AsyncTask<String, Void, Bitmap>
  • String: tipo definito in doInBackground.
  • Void: tipo di ritorno di onPreExecute.
  • Bitmap: valore di ritorno di doInBackground.

Geolocalizzazione
...

I sistemi di geolocalizzazione più comuni sono:

  • network-based: fa riferimento all'indirizzo IP, rileva WiFi e GSM della zona ed in base a questi calcola la propria posizione. Il GMS non è molto accurato ma immancabile nei dispositivi.
  • GPS (Global Positioning System): si basa sull'intercettazione di messaggi inviati da satelliti che ruotano attorno alla Terra.
    Si richiama il LocationManager per ottenere tutti i valori riguardanti la posizione.
    Bisogna prima implementare l'interfaccia LocationListener, che prevede tre metodi:
  • onStatusChanged
  • onProviderEnabled
  • onProviderDisabled
    Questi metodi notificano, rispettivamente, se il provider è disponibile o meno e se è abilitato o no.
Nota

In Java, le callback si passano definendo degli oggetti che implementano una determinata interfaccia.
