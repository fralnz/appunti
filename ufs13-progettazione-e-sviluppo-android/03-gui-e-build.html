  • altri...

Eventi
...

Build
...


Gli apk sono degli zip rinominati. All'interno di un apk sono contenuti:

  • AndroidManifest.xml: contiene tutti i riferimenti di schermate disponibili con permessi annessi.
  • resources.arsc: file della grafica compilata. Contiene file come immagini, font, video, audio...
  • file .dex: bytecode cifrato. Le nostre applicazioni devono essere sempre firmate, anche in modalità debug.
  • risorse decompilate.
    Una volta decompressi, gli apk vengono installati con ADB (Android Device Build).

Definizioni e considerazioni
...

  • aapt: Android Asset Packaging Tool: Legge gli xml e genera R.java
  • aidl: Android Interfce Definition Language, converte interfacce di servizi .aidl in interfacce Java
  • Tutto il codice Java è compilato generando bytecode.class.
  • dex: converte i .class in file .dex eseguibili da Dalvik/ART
  • apkbuilder: comprime e impacchetta i .dex e le risorse in un unico file .apk.
  • jardesigner: permette di inserire una firma nell'apk.
  • zipalign: consente di ottimizzare le risorse di memoria utilizzata dall'applicazione di un dispositivo.

Firma
...

Ogni applicazione Android ha bisogno di una firma privata che identifichi l'autore per essere pubblicata, generando quindi fiducia.

Nota

Non rilasciare mai le applicazioni in debug mode! Si può fare il reverse engineering sull'apk ed ottenere i file sorgenti.

Sviluppo
...

Noi non scriveremo mail il main, in quanto già definito all'interno dell'Application Framework.
