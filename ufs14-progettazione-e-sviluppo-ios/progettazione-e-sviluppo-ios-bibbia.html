Sistema Operativo
...

MacOS
...

L'architettura di MacOS è divisa in livelli:

  • User Experience: il livello più vicino all’utente.
  • Application Frameworks: dove si trovano le API di Carbon, UIKit e SwiftUI.
  • Core Frameworks: livello per incorporare nelle app la grafica 2D e 3D, animazioni, effetti per le immagini, funzionalità audio e video di livello professionale.
  • Darwin: parte fondamentale (Open Source) del sistema operativo.

Il kernel xnu è “ibrido”, formato da prevalentemente da due tecnologie, Mach e BSD in aggiunta ad una componente fondamentale: IOKit, che è una raccolta di framework di sistema, librerie, strumenti e altre risorse per la creazione dei driver in macOS.

  • Mach: fornisce l’ambiente per la memoria virtuale.
  • BSD: fornisce le interfacce per il filesystem e le chiamate di sistema.
  • IOKit: gestisce le periferiche.

iOS
...

L'architettura di iOS è la stessa di MacOS per quanto riguarda i livelli inferiori:

  • Cocoa Touch: per sviluppare applicazioni per iOS.
  • Media: per comprendere, accedere e sviluppare applicazioni e servizi mobili su dispositivi iOS che forniscono interfacce grafiche, animazioni e accesso agli elementi visivi nativi del dispositivo e al comparto audio.
  • Core Services: fornisce servizi essenziali per le app, senza avere nulla a che vedere con la loro interfaccia grafica.
  • Core OS: livello più basso, si interfaccia direttamente con l'hardware. Include servizi come networking di basso livello, accesso ad accessori esterni e i soliti servizi fondamentali del sistema operativo come la gestione della memoria, gestione del filesystem e dei thread.

SwiftUI
...

SwiftUI è un linguaggio dichiarativo. Fa infatti uso delle closure (funzioni anonime) e protocolli (interfacce).

Dichiarazioni
...

Variabili:

var variabile: tipo = "valore"

Costanti:

let costante: tipo = "valore"

Funzioni:

func riordina(numeriPresentiIn: inout[Int], inModalità: String) -> [int]{
	numeriPresentiIn.sort()
	if inModalità == "desc" {
		numeriPresentiIn.reverse()
	}
	return numeriPresentiIn
}

var elenco = [3, 4, 1]
print(riordina(numeriPresentiIn: &elenco, inModalità: "desc"))

Condizioni
...

Per rappresentare le condizioni si usa if come negli altri linguaggi. Le graffe sono sempre obbligatorie.

var optionalString: String? = "Hello"
print(optionalString == nil) //false

Nel caso in cui un oggetto non esista (sia nil) i suoi metodi non vengono chiamati:

var prova: Cane = nil
prova?.abbaia()

Opzionalità negli array:

var prova: [Int?] = [1, nil, 3] //prova può contenere elementi che possono esistere o no
var prova: [Int]? = [1, nil, 3] //qui da errore: prova o è nil o contiene elementi, ma essi non possono essere nil
var prova: [Int?]? = [1, nil, 3]

Dizionari
...

Sono definizioni chiave-valore:

var occupations=[
	"Antonio": "ErMejo",
	"Giuseppe": "Meccanico",
]
occupations=["Giorgio"] = "Giornalista"

Un modo alternativo è:

var dict: Dictionary<Float, Int> = Dictionary<Float, Int>()
var dict: [Float, Int] = Dictionary<Float, Int>()
// quindi:
var dict: [String!, Int?] = {
	"Antonio": 32,
	"Mario": 54
}

Cicli
...

let IndividualScores=[75, 43, 103, 87, 12]
var teamScore = 0
for score in individualScores {
	if score > 50 {
		teamScore += 3
	} else {
		teamScore +=1
	}
}
print(teamScore)
// dovrebbe dare 13

Un altro metodo per gestire i valori opzionali è fornire un valore predefinito utilizzando l'operatore ??: se manca il valore facoltativo, viene utilizzato il valore di fallback.

let nickname: String? = nil
let fullName: String = Antonio
let informalGreeting = "Hi \(nickname ?? fullName)"

Switch:

let vegetable = "red pepper"
switch vegetable{
	case "celery":
		print("Add some raisins...")
	case "cucumber", "watercress":
		print("that would make a good tea sandwich")
	case let x where x.hasSuffix("pepper"):
		print("that's some spicy \(x)")
}

Do-while non esiste ma si chiama repeat:

repeat{
	m *= 2
} while m < 100

var total = 0
for i in 0..4{
	total += i
}

Costruttori
...

Per creare un'istanza di una classe non deve fare uso di new come in Java, ma si deve comunque chiamare il costruttore:

var shape = Shape()
shape.numOfSides = 7
var shapeDescription=shape.simpleDescription()

Per definire il costruttore si usa init.
In Swift non esiste il Garbage Collector, ma esiste ma l'ARC. Non esistendo il Garbage Collector, esiste la possibilità di definire un deinit per delle classi per liberare lo spazio in memoria.
In una costante (definita con let), posso cambiare parametri di un oggetto ma non posso cambiare oggetto (quindi l'indirizzo in memoria).
Ogni proprietà di un oggetto ha un metodo get e set. Quindi:

var perimeter: Double{
	get {
		return 3.0 * sideLength
	}
	set {
		sideLength = newValue / 3.0
	}
}
var t = Triangle(sideLen: 3.1, name: "Triangle")
print(t.perimeter)    // stampa 9.3
t.perimeter = 9.9
print(t.sideLength)   // stampa 3.3

Si possono usare anche le proprietà willSet (quando deve essere modificata) e didSet (quando è già stata modificata).

var _test: Int = 0 {
	willSet{
		println("Old value is \(_test), new value is \(newvalue)")
	}
	didSet{
		println("Old value is \(oldValue), new value is \(_test_)")
	}
}

Protocolli
...

I protocolli sono l'equivalente delle Interfacce in Java.

protocol ExampleProtocol {
	var simpleDescr: String { get }
	mutating func adjust()
}

Estensioni
...

Si possono estendere sul momento oggetti, aggiungendo metodi alla loro classe:

var prova: Int = 45
extension Int{
	func convBinario() -> String {
		// ...
	}
}

Adjust
...

La funzione adjust è comunemente usata per "riaggiustare" dei dati:

mutating func adjust() {
	self += 42;
}

Closure
...

func greet (person: String, day: String) -> String {
	return "Hello \(person), today is \(day)."
}
greet(person: "Bob", day: "Wednesday")

Si possono definire funzioni come variabili, chiamata closure:

var greet = {
	print("Hello, World!")
}

greet()

Le closure possono anche avere parametri:

let greetUser = { (name: String) in
	print("Hey there, \(name).")
}

greetUser("Paolo")
Nota

in segnala che è finita la dichiarazione dei parametri ed inizia il codice della closure

Si possono creare dei costruttori per le closure, oppure le annotazioni:

func makeIncrementer() -> ((Int) -> Int) {
	func addOne(number: Int) -> Int{
		return 1 + number
	}
	return addOne
}
var increment = makeIncrementer()

Si possono passare closure come argomenti di funzioni (simile alle callback):

func hasAnyMatches(list: [Int], condition: (Int) -> Bool) -> Bool {
	for item in list {
		if condition(item) {
			return true
		}
	}
	return false
}
func lessThanTen (number: Int) -> Bool {
	return number < 10
}
var numbers=128, 19, 7, 12]
hasAnyMatches(list: numbers, condition: LessThanTen)

Anche il metodo map accetta closure come argomenti:

let sortedNumbers = numbers.sorted { $0 > $1 }

Strutture
...

Le strutture (struct) sono come le classi, ma senza il concetto di ereditarietà e polimorfismo.
Inoltre le strutture vengono passate per valore, anziché per indirizzo come nelle classi. Ciò comporta un maggior uso di memoria, ma anche una maggiore ottimizzazione: appena finisce lo scope di una struttura, quest'ultima viene completamente eliminata dalla memoria.

Enum
...

Si usa enum per definire un'enumerazione. Di solito assume solo valori numerici:

enum Rank: Int{
	case ace=1
	case two, three, four, five, six, seven, eight, nine, ten
	case jack, queen, king
	func simpleDescription() -> String{
		switch self {
		case .ace:
			return "ace"
		case .jack:
			return "jack"
		case .queen:
			return "queen"
		case .king:
			return "king"
		default:
			return String(self.rawValue)
		}
	}
}

let ace=Rank.ace
let aceRawValue = ace.rawValue
if let convertedRank = Rank(rawValue: 3){
	let threeDescription=convertedRank.simpleDescription()
	print(threeDescription)
}
Nota

rawValue corrisponde al tipo della struttura e al valore assegnato (se la struttura è di tipo Int e il valore è one, rawValue sarà 1)

Eccezioni
...

Per rappresentare gli errori si utilizza il protocollo Error:

enum PrinterError : Error {
	case outOfPaper
	case noToner
	case onFire
}

Per sollevare un eccezione si usa Throws.
Esempio di try-catch:

do {
	let printerResponse = try send(job: 1440, toPrinter: "Gutenberg")
	print(printerResponse)
} catch PrinterError.onFire{
	print("on fire!")
} catch let printerError as PrinterError{
	print("Error: /(printerError)")
} catch{
	print(error)
}

Un modo alternativo e migliore per gestire le eccezioni si usa:

let printerSuccess = try? send(job: 1884, toPrinter("Gutenberg"))
let printerFailure = try? send(job: 1885, toPrinter("Never Has Toner"))

Contenitori
...

Le viste possono essere organizzate in contenitori e possono essere organizzate in una direzione. Questo tipo di vista prende il nome di Stack.
Ci sono tre tipi di stack:

  • VStack (Vertical Stack)
  • HStack (Horizontal Stack)
  • ZStack (Depth Stack)
    Ogni contenitore può contenere più componenti, che siano viste o altri contenitori.
    Come in CSS, lo stile del padre viene ereditato dai componenti figli.
    Si fa utilizzo di una programmazione dichiarativa.

Viste
...

Le viste in SwiftUI sono delle strutture conformi al protocollo View. Bisogna implementare la proprietà body:

struct BasicSyntax: View {
	var body: some View {
		// Stampa a video il testo
		Text("Hello World!")
	}
}

Si usa la parola chiave some per definire un tipo di oggetto opaco, cioè non chiaro: definiamo quindi un componente che appartiene alla categoria delle View (per simulare il concetto di ereditarietà).

Modificatori
...

Per modificare il modo in cui si visualizzano gli elementi in una View si usano i modificatori. Per esempio: largeTitle, title, title2, title3...

Text("Colore giallo)
	.font(.system(size: 45, weight: .light))
Nota

Si scrive .system senza niente prima perché nella definizione della classe si sa per certezza che si ha solo un oggetto di tipo Font (definito con _)

Componenti
...

Si possono creare componenti riutilizzabili dichiarandoli come strutture:

struct ImageButton: View {
	@State var premuto : bool = true
	var iconFill: String = "car.fill"
	var iconNoFill: String = "car"
	var text: String = "premuto"
	var textNoFill: String = "non premuto"
	var color: Color?
	var body: some View {
		Button(action: {
			premuto.toggle()
		}) {
			Image(systemName: premuto ? iconFill : iconNoFill)
			Test(premuto ? text : textNoFill)
		}
		.frame(width: 150)
		.padding()
		.background(Color.red)
		.cornerRadius(20)
	}
}

State
...

Le classi @State vanno definite all'interno della struttura ma al di fuori dei body.

@State var premuto : bool = true

Se vogliamo usare @State dalla funzione chiamante e non dichiararli nel componente usiamo il two-way binding:

@Binding var state: Bool

Slider
...

Lo Slider è una barra che va da 0 a 1, in cui l'utente sposta il cursore, impostando il valore.
Il valore minimo e massimo possono essere assegnati manualmente, basta che siano di tipo numerico.

@State private var sliderValue = 0.5
@State private var age = 18.0
Slider(value: $sliderValue)
Slider(valie: $age, in: 1...100, step: 1)
	.padding(horizontal)

Stepper
...

Lo Stepper è uno slider con pulsanti, permettendo di modificare il valore per step:

Stepper("Inserisci età", value: $age, in: 0...130)
// oppure
Stepper("Inserisci età", inIncrement: {
	age +=1
}, onDecrement: {
	age -= 1
})
Text("La tua età è \(age)")

Picker
...

Un componente grafico che funge da "selezione" di elementi.
Questo componente prevede che venga dichiarate una variabile @State.
Come ultimo parametro abbiamo a disposizione una closure, in cui bisogna mettere un ciclo ForEach che itera su tutti gli elementi del Picker

var years 0 Array(2000...2021)
@State private var selectedYear = 2021
Picker("Pick a year", selection: $selectedYear) {
	ForEach(years, id: \.self) {
		Text("\($0.formatted(.number.grouping(.never)))")
	}
}.pickerStyle(WheelPickerStyle())

Abbiamo bisogno di un id in quanto ogni elemento del Picker devono essere univoci (spesso si usa un hash).

Liste
...

Un insieme di elementi scorrevoli, anche diversi tra loro, ma che devono restituire una vista.

struct List_MoveRow: View {
    @State private var data = ["Hit the Edit button to reorder", "Practice Coding",
                                "Grocery shopping", "Get tickets", "Clean house",
                                "Do laundry", "Cook dinner", "Paint room"]
    
    var body: some View {
        NavigationStack {
            List {
                ForEach(data, id: \.self) { datum in
                    Text(datum).font(.system(size: 24)).padding()
                }
                .onMove(perform: moveRow)
            }
            .navigationBarTitle(Text("To Do"))
            .navigationBarItems(trailing: EditButton())
            .accentColor(.green)
        }
    }
    
    func moveRow(from indexes: IndexSet, to destination: Int) {
        if let first = indexes.first {
            data.insert(data.remove(at: first), at: destination)
        }
    }
}

struct Restaurant: Identifiable {
    let id = UUID()
    let name: String
}

struct RestaurantRow: View {
    var restaurant: Restaurant
    
    var body: some View {
        Text("Come and eat at \(restaurant.name)")
    }
}

struct ContentView: View {
    let restaurants = [
        Restaurant(name: "Sushiamo"),
        Restaurant(name: "Ma-Kebap"),
        Restaurant(name: "Lino's Pizza")
    ]
    
    var body: some View {
        List(restaurants) { restaurant in
            RestaurantRow(restaurant: restaurant)
        }
    }
}

Il pulsante EditButton() è una funzione incorporata che restituisce una vista (Button) che attiverà automaticamente la modalità di modifica nell’elenco. Il suo testo dice "Modifica" e quando toccato appare le maniglie di spostamento sulle righe ed il testo del pulsante
dice "Fatto".

Navigazione
...

NavigationLink è un componente usato per spostarsi da una schermata all'altra.
Si usa la NavigationBar come barra di navigazione al di spora dell'applicazione. Si possono impostare immagini, titolo e molto altro.

NavigationStack {
	VStack(spacing: 25) {
		Image(systemName: "globe").font(.largeTitle)
		Text("NavigationView")
		//...
		Spacer()
	}.font(.title)
	.padding(.top, 25)
	.navigationBarTitle(Text("Navigation Bar Items”),
		displayMode: .inline)
	// Add some bar items to the nav bar
	.navigationBarItems(
		// Button on the leading side
		leading: Button(action: { }) {
			Image(systemName: "bell.fill")
		}.accentColor(.pink)
		// Button on the trailing side
		, trailing: Button("Actions", action: { })
			.accentColor(.pink)
	)}
}

ScrollView
...

La ScrollView ingloba l'HStack o il VStack e mostra una lista a scorrimento.
Parametri:

  • orizzontale o verticale
  • barra di scorrimento
  • closure
    Le Liste sono ScrollView fatte di componenti uguali.
    ScrollViewReader è un proxy che prende l'istanza del primo oggetto iterato.

	.ignoreSafeArea(.all)

Servizi Web
...

Per usare le API di servizi web bisogna definire classi modello con protocollo Identifiable (con ID) e Codable (consente la serializzazione / deserializzazione in JSON).
La classe URLSession mette a disposizione un singleton con metodo dataTask, che ha come parametri l'url e una closure per gestire i dati ricevuti.
request è un oggetto di richiesta URL.
completeHandler è il gestore di completamento che viene invocato quando la richiesta viene completata. I suoi parametri sono:

  • data: i dati restituiti dal server.
  • response: un oggetto che fornisci i metadati della risposta, come le intestazioni HTTP e il codice di stato.
  • error: un oggetto che indica l'errore.
    Dopo aver creato l'attività dobbiamo invocare il metodo resume().
    Di default, l'API URLSession è asincrona.
    Esempio per caricare immagini:
func loadImage(for urlString: String) {
	guard let url = URL(string: urlString) else { return }

	let task = URLSession.shared.dataTask(with: url) { data, response, error in
		guard let data = data else { return }
		DispatchQueue.main.async {
			self.image ? UIImage(data: data) ?? UIImage()
		}
	}
	task.resume()
}

Mappe
...

I permessi dell'applicazione vanno salvati nel file info.plist (il corrispettivo di manifest.xml di Android).
Per visualizzare una mappa di usa MapView.

struct ContentView: View {
	@State private var region = MKCoordinateRegion(
	center: CLLocationCoordinate2D(latitude: 45.563487, longitude: 9.198354),
	span: MKCoordinateSpan(latitudeDelta: 0.03, longitudeDelta: 0.03)
	)
	var body: some View {
		Map(coordinateRegion: $region)
	}
}

User Defaults
...

Gli User Defaults sono le impostazioni dell'utente, equivalenti agli Shared Preferences di Android.
Per salvare e caricare gli User Defaults si usa il decoratore @AppStorage:

@AppStorage("tapCountKey") private var tapCount = 0

Per il resto funziona esattamente come per gli @State.
