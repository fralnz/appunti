Java nasce per supportare il paradigma a oggetti (OOP, Object Oriented Programming).
La OOP si basa su:

  • Classe: definizione di un determinato tipo di oggetto.
  • Oggetto: istanza della classe.
    Esempio: "Macchina" è la classe generica, "Nissan", "Audi" e "BWM" sono oggetti.

Classe
...

Una classe è una componente logica che permette di definire proprietà e funzionalità comuni a tutti gli oggetti di un tipo, fornendo una struttura di riferimento per creare specifici oggetti di quel tipo. Una classe, quindi, non richiede allocazione di memoria.
Quando si crea una classe, si definisce un tipo di variabile.
Sintassi:

public class NomeClasse { ... }

Per convenzione di usa il PascalCase.
Una classe definisce:

  • Variabili: i valori che rappresentano lo stato di un oggetto.
  • Metodi: sequenze di istruzioni che rappresentano comportamenti o azioni che gli oggetti possono eseguire.
    Variabili e metodi legati ad un oggetto si definiscono "di istanza".

Variabili
...

Ci sono tre tipi di variabili:

  • Locali: definite all'interno di un blocco.
  • Di istanza: definite all'interno di una classe ma all'esterno di qualsiasi metodo.
  • Statiche (o di classe): simili a variabili globali, comuni a tutti i metodi.
class A{
	String s2;         //Variabile di istanza
	int j;             //Variabile di istanza
	static String s1;  //Variabile statica
	static int i;      //Variabile statica
}

Le variabili di istanza o statiche possono non essere inizializzate: al momento della creazione dell'oggetto, l'interprete le inizializza in automatico con il valore di default.
NBV: il valore di default dei booleani è false, mentre per gli oggetti è null.
La keyword final si utilizza per inizializzare costanti.

Differenze tra variabili statiche e di istanza
...

  • Creazione: le variabili di istanza vengono allocate alla creazione dell'oggetto e deallocate alla distruzione dell'oggetto. Le variabili statiche vengono create al caricamento in memoria della classe e distrutte a termine del programma.
  • Modifiche: le modifiche per una variabile statica si riflettono anche per gli altri oggetti, mentre le modifiche apportate su una variabile di istanza influenzano sono l'oggetto.

Metodi
...

I metodi si dividono in:

  • Metodi di istanza: vengono invocati su specifici oggetti e dello stesso tipo della classe in cui il metodo è definito.
  • Metodi statici: non hanno alcuna relazione con un oggetto. Vengono dunque utilizzati per funzionalità di utilità generale, che non necessitano di informazioni su un particolare oggetto. E' buona convenzione raggrupparli tutti in una stessa classe che riunisce i metodi di utilità varia e generale
public class ContoCorrente
	String codiceCliente;
	String tipoConto;
	double saldo;
	static String nomeBanca = "Banca Rizzoli";
	static final double tassoInteresse = 0.03;
	static int numeroStampe;
	public void stampaConto () {
		System.out.println("Banca: " + nomeBanca);
		System.out.printin("Codice cliente: " + codiceCliente);
		System.out.println("Tipo conto: " + tipoConto);
		System.out.println("Saldo: " + saldo);
		numeroStampe++;
	}
	public static void stampaMenù () {
		System.out.println("1 - Versamento");
		System.out.println("2 - Prelievo");
		System.out.printin("3 - Visualizzazione saldo");
		System.out.println("4 - Terminazione programma");
		numeroStampe++;
	}
}

Oggetti
...

Gli oggetti, anche detti istanze di una classe, sono il concetto base della OOP. Sono entità fisiche cui corrispondono a specifici oggetti fisici o astratti del mondo reale, creati a partire dal "template" definito da una classe.
Una classe può istanziare nessuno, uno o più oggetti. Un oggetto di una determinata classe è una variabile di quel tipo. Dunque, al contrario delle classi, ogni oggetto richiede una parte di memoria allocata, in particolare per memorizzare i valori delle variabili di istanza.
Un oggetto viene definito da:

  • Identità: un identificatore, univoco tra gli oggetti di una stessa classe, che ci permette di accedere a esso.
  • Stato: le caratteristiche e le proprietà che lo definiscono, ovvero le variabili di istanza.
  • Comportamento: le funzionalità che gli permettono di mutare il proprio stato e di interagire con altri oggetti, ovvero i metodi di istanza.

Creazione di oggetti
...

Una volta che si è definita una classe, si possono istanziare degli oggetti di tale classe, sia all'interno della medesima classe che, come più di frequente, all'interno di altre classi.
Sintassi:

NomeClasse nomeOggetto = new NomeClasse ();

Così come per gli array, la creazione può dunque essere suddivisa in due differenti fasi:

  • Dichiarazione: viene creata una variabile vuota che conterrà il riferimento all'oggetto.
NomeClasse nomeOggetto;
  • Allocazione: tramite la keyword new, viene allocata la memoria necessaria per istanziare l'oggetto (principalmente per le variabili di istanza) e viene riempita la variabile precedentemente creata con il riferimento all'indirizzo di memoria in cui sarà salvato l'oggetto.
nomeOggetto = new NomeClasse();

Esempio:

public class DemoContoCorrente {
	public static void main (String [] args) {
		ContoCorrente contoMario = new ContoCorrente ();
		ContoCorrente contoGiovanni = new ContoCorrente ();
		ContoCorrente contoEstero = new ContoCorrente ();
	}
}

Accesso esterno a variabili e metodi
...

Variabili e metodi di istanza sono accessibili solo tramite un oggetto, valorizzando le variabili di istanza con i valori di tale oggetto, mentre variabili e metodi statici sono accessibili senza la creazione di un oggetto.
Per accedere a dati di una classe da una classe differente, le sintassi sono le seguenti:

  • Accesso a variabili statiche: NomeClasse.nomeVariabile;
  • Accesso a metodi statici: NomeClasse.nomeMetodo([argomenti]);
  • Accesso a variabili di istanza: nomeOggetto.nomeVariabile;
  • Accesso a metodi di istanza: nomeOggetto.nomeMetodo([argomenti]);
    Nel caso di metodi e variabili di istanza, nomeOggetto è detto oggetto chiamante.
public class ContoCorrente {
	String codiceCliente;
	String tipoConto;
	double saldo;
	static String nomeBanca = "Banca Rizzoli";
	static final double tassoInteresse = 0.03;
	static int numeroStampe;
	public void stampaConto () { ... }
	public static void stampaMenù () { ... }
	public double raddoppiaConto( ) {
		return saldo * 2;
	}
}

E' possibile accedere da una classe differente a una variabile o a un metodo statico tramite un oggetto di quella classe, ma non è consigliato: potrebbe essere poco chiaro a livello di leggibilità data la scorrettezza logica.
In fase di definizione di un metodo di istanza, in esso si possono utilizzare sia variabili statiche che variabili di istanza definite per gli oggetti di quella classe: le variabili di istanza vengono valorizzate con i valori dell'oggetto chiamante.
Nei metodi statici, invece, si possono utilizzare solo variabili statiche, non a variabili di istanza: senza un oggetto che invochi il metodo, non esiste una copia delle variabili di istanza cui fare riferimento.
Le considerazioni fatte per le variabili valgono anche per i metodi: un metodo di istanza può invocare sia metodi statici che di istanza, mentre un metodo statico può invocare solo metodi statici.

Costruttori
...

Quando si crea un oggetto di una classe utilizzando l'operatore new, si invoca automaticamente un particolare tipo di metodo chiamato costruttore. In tale metodo si può definire qualsiasi logica si voglia, ma tendenzialmente il suo scopo è compiere operazioni di inizializzazione delle variabili di istanza e di verifica sulla correttezza di tali valori.
Non è possibile utilizzare un oggetto già esistente per chiamare un costruttore: vengono chiamati solo al momento della creazione.
Per definire un metodo costruttore, è sufficiente che tale metodo abbia lo stesso nome della classe utilizzata per creare tali oggetti, maiuscola inclusa. Inoltre, i costruttori richiedono solo il modificatore di accesso, non va indicato il tipo di ritorno o la keyword void.
Nel caso in cui non si indichino metodi costruttori, Java fornisce di default il costruttore senza parametri che inizializza tutte le variabili di istanza ai valori di default. Nel caso in cui venga definito esplicitamente anche un solo costruttore, anch'esso sia con dei parametri, Java non fornisce tale costruttore di default.

Tipi di costruttori
...

Esistono tre tipi di costruttori:

  • Costruttore senza parametri (o di default): le variabili di istanza dell'oggetto vengono inizializzate ai valori di default dei corrispondenti tipi. Nonostante questo venga già fatto automaticamente, è consigliabile effettuare tale inizializzazione esplicitamente al fine di rendere il codice più leggibile.
  • Costruttore parametrizzato: tale costruttore accetta il passaggio di argomenti attraverso le parentesi tonde poste al termine dell'istruzione di creazione di un oggetto. In tal caso, le variabili di istanza vengono inizializzate con i valori passati come argomento.
  • Costruttore copia: tale costruttore crea un oggetto a partire da un altro oggetto della stessa classe, andando a copiare i valori delle variabili di istanza di tale oggetto nelle variabili di istanza di quello che si sta creando.

Overloading di costruttori
...

Risulta utile, per una medesima classe, andare a definire più costruttori: data la necessità che i costruttori abbiano lo stesso nome della classe, avremmo quindi più metodi con il medesimo nome ma differente lista di parametri.
Viene dunque effettuato l'overloading del metodo, concetto già affrontato nelle lezioni precedenti. Alla creazione dell'oggetto, l'interprete andrà a ricercare il miglior match tra gli argomenti passati e la lista di parametri definiti nei vari costruttori.
Generalmente i costruttori sono raggruppati e posti come primi metodi di una classe.

Keyword this
...

Parametri e variabili di istanza possono anche avere lo stesso nome. Questo è possibile perché quando invochiamo un metodo di istanza esiste un riferimento implicito al nome dell'oggetto chiamante: tale riferimento è dato dalla keyword this.
Nel caso in cui l'unica variabile con quel nome visibile a quel metodo sia la variabile di istanza, tale nome può essere utilizzato per riferirsi alla variabile di istanza senza l'uso della keyword this, che diventa opzionale.
Nel caso in cui questo non sia invece vero, dunque il nome di un parametro e il nome di una variabile di istanza siano lo stesso, avviene ciò che è definito variable shadowing:

  • Utilizzare solo l'identificativo permette di accedere al parametro: la variabile dichiarata in uno scope più interno "fa ombra" sulla variabile dichiarata nello scope più esterno, coprendola.
  • Utilizzare this.identificativo permette di accedere alla variabile di istanza dell'oggetto chiamante (non il parametro).

Constructor chaining
...

Così come un metodo può invocare altri metodi, anche un costruttore può invocare altri metodi definiti nella classe, così come può invocare anche altri costruttori: quest'ultimo caso è detto constructor chaining.
Questo viene fatto tramite la keyword this fatta seguire da una coppia di parentesi, contenenti eventuali argomenti: tale istruzione corrisponde all'invocazione del metodo costruttore della medesima classe i cui parametri matchino con gli argomenti passati.
E' necessario che, se un costruttore voglia utilizzare tale istruzione, essa sia la primissima del suo corpo.
Questa tecnica viene utilizzata tendenzialmente per indicare la logica dei controlli sui valori delle variabili unicamente nel costruttore che inizializzi tutte le variabili di istanza: tutti gli altri costruttori, incluso quello di default, consistono semplicemente nel richiamare tale costruttore passando i letterali di default per gli argomenti non previsti da quel costruttore.
Nel caso in cui un valore non sia valido, per il momento ci limitiamo a effettuare la chiamata System.exit(0), che termina il programma, oppure a settare la variabile di istanza con il suo valore di default stampando un messaggio di errore.
