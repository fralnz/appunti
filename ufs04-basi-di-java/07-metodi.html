Cos'è un metodo
...

Un metodo raggruppa una sequenza di istruzioni che realizzano una certa funzionalità, assegnando loro un identificatore. Tale sequenza può essere eseguita ogni volta che è necessario, semplicemente riferendosi a esse tramite l'identificatore.
Un metodo, quindi, non è altro che una porzione di codice isolata e riutilizzabile.

Definizione e invocazione
...

Per utilizzare un metodo, sono necessarie due differenti fasi:

  • Definizione: il metodo viene creato in modo tale che sia pronto per essere utilizzato. E' necessario definire:
    • Intestazione del metodo, nella quale vengono riportati i modificatori di accesso, l'identificatore e gli eventuali parametri (di cui si parlerà in seguito).
    • Corpo del metodo, ovvero l'insieme di istruzioni che lo compongono. Esse vengono poste tra graffe, definendo dunque uno scope a parte nel quale le variabili dichiarate all'interno sono locali.
  • Invocazione (o richiamo): vengono eseguite tutte le istruzioni che compongono il metodo e, al termine, il controllo torna nella posizione in cui era stata effettuata tale invocazione.
    Si osservi, dunque, che un metodo definito ma mai invocato non viene in realtà mai eseguito.

Tipi di metodi
...

Java ha due tipi di metodi:

  • Metodi che restituiscono un valore. Al contrario di altri linguaggi, in Java essi non vengono chiamati funzioni perché sono comunque legati a una classe.
  • Metodi che non restituiscono nessun valore: questi metodi sono detti void.
    Altre classificazioni riguardanti i metodi saranno viste quando si parlerà di programmazione a oggetti. Per il momento, si dia per assodato che i metodi che scriveremo in questa lezione sono pubblici (public) e statici (static), modificatori di accesso necessari per la definizione di entrambe le tipologie di metodi viste.
    In generale, tutti i metodi vengono necessariamente definiti all'interno di una classe: si dice che essi appartengono alla classe in cui sono definiti.

Sintassi
...

Sintassi dei metodi void:

public static void nomeMetodo ([tipoParam1 param1, …, tipoParamN paramN]) {
	corpoMetodo;
}

Per invocare un metodo void, invece, occorre semplicemente riportare il suo identificatore nel codice in cui si vogliano eseguire le istruzioni di tale metodo, con gli eventuali parametri richiesti tra parentesi.
Esempio:

public class MetodiVoid {
	public static void main(String[] args) {
		System.out. println("Sono nel main prima dell'invocazione del metodo!");
		stampaDaMetodo( );
		System.out.println("Sono nel main dopo l'invocazione del metodo!");
		stampaDaMetodo( );
	}
	public static void stampaDaMetodo( ) {
		System.out.println("Sono in un metodo!");
	}
}
/* Output:
Sono nel main prima dell'invocazione del metodo!
Sono in un metodo!
Sono nel main dopo l'invocazione del metodo!
Sono in un metodo!

Sintassi del metodo che ritorna un valore:

public static tipoValoreReturn nomeMetodo ([tipoParam1 param1, …, tipoParamN paramN]) {
	corpoMetodo;
	return espressione;
}

Per invocare un metodo che restituisce un valore si segue lo stesso procedimento per i void, ma tale invocazione deve essere posta in una qualsiasi posizione in cui si potrebbe usare un elemento dello stesso tipo di ritorno del metodo (espressioni, assegnamenti, stampe a schermo, parametri di altri metodi, …).

public class MetodiReturn {
	public static void main(String[] args) {
		int area = areaQuadratoLato2();
		area = area + areaQuadratoLato2();
		System.out.println("L'area è " + areaQuadratoLato2());
	}
	public static int areaQuadratoLato2( ) {
		int lato = 2;
		return lato * lato;
	}
}
/* Output:
L'area è 4

Return
...

La keyword return viene utilizzata obbligatoriamente nei metodi che ritornano un valore, seguita da un letterale, una variabile, una costante o un'espressione del tipo richiesto per indicare tale valore di ritorno.
Solitamente, tale parola chiave viene indicata una sola volta per metodo come ultima istruzione. Tuttavia, tale soluzione è utilizzata per migliorare la leggibilità, ma si potrebbero indicare anche più istruzioni di return.

Parametri
...

Le variabili che vengono predisposte nella definizione del metodo per ricevere i valori passati dal chiamante sono detti parametri; il valore effettivo che il parametro assume al momento dell'invocazione è detto argomento. Al momento dell'invocazione, dunque, è come se avvenisse l'assegnamento al parametro posto nella definizione del metodo.
Gli argomenti passati corrispondono ai parametri dell'intestazione del metodo in:

  • Numero,
  • Ordine,
  • Tipo.
    Es 1

    Es 2

Passaggio per valore e riferimento
...

Quando si effettua il passaggio di un argomento attraverso l'uso di una variabile, il passaggio può essere per:

  • Valore: una modifica nel metodo non ha ripercussioni sul chiamante.
  • Riferimento (indirizzo di memoria): una modifica del metodo ha ripercussioni sul chiamante.
    Caso particolare sono le stringhe, variabili di tipo classe che sono però immutabili, quindi eventuali modifiche nel metodo non hanno effetto sulla stringa presente nel chiamante.
    Per quanto riguarda le variabili indicizzate di un array, il loro uso come argomento è il medesimo di una variabile di quel tipo, come visto finora.
    Per quanto riguarda invece interi array, essi vengono utilizzati come segue:
  • Nell'intestazione, si pone una coppia di parentesi quadre tra il tipo del parametro e il suo nome.
  • Nell'invocazione, viene passato tramite il suo identificatore, senza l'uso di parentesi quadre.
  • Nel tipo di ritorno di un metodo che ritorna un array, il tipo va fatto seguire da parentesi quadre.
  • Nel return, anche in questo caso viene indicato l'identificatore dell'array, senza parentesi quadre. Un metodo che ritorna un array può essere invocato in qualsiasi posizione in cui possa essere usato un array di quel tipo.

Overloading dei metodi
...

In Java, è possibile avere all'interno della stessa classe più metodi che hanno lo stesso identificativo: si dice che si sta effettuando l'overloading (sovraccarico) del metodo.
Questo è possibile perché in Java non è l'identificatore di un metodo a dover essere univoco, ma la sua signature (firma): essa è composta dalla combinazione dell'identificativo e della lista dei parametri che esso richiede, inteso come numero, ordine e tipo di essi.
Si osservi come il tipo di ritorno non fa parte della firma di un metodo, quindi non possiamo creare due metodi che differiscono solo per questo aspetto.
In caso di overloading di un metodo, il compilatore riesce a identificare quale metodo eseguire sulla base del match tra gli argomenti passati in fase di invocazione e la lista dei parametri dell'intestazione del metodo.

Problemi con l'overloading
...

Nel caso in cui non esiste alcun metodo che rispetti tale corrispondenza precisa, il compilatore prova a verificare la presenza di corrispondenza rispettate tramite l'uso di casting implicito. Si presti attenzione al fatto che questo potrebbe condurre a risultati inattesi.
Nel caso in cui anche tali valutazioni non restituiscano alcuna corrispondenza, verrà restituito un errore.
Date queste situazioni potenzialmente inattese, quando possibile è preferibile evitare l'overloading e usare nomi dei metodi differenti, il più descrittivi possibile. Vedremo come in alcuni casi la soluzione dell'overloading sia comunque necessaria.

Numero di parametri variabile
...

Tramite overloading è quindi possibile definire metodi con la stessa funzionalità ma che funzionino con un numero di parametri differenti. Tuttavia, secondo questo approccio, sarebbe necessario conoscere il numero di argomenti passati o andare a definire un numero elevato di metodi con la stessa funzionalità.
Java permette di definire metodi che accettano un numero variabile di argomenti, purché siano dello stesso tipo, organizzandoli in un array inizializzato con i valori passati come argomenti e con lunghezza pari al numero di essi. Tale specifica di parametro è detta vararg.
La sintassi risulta essere dunque la seguente:

public static tipoValoreReturn nomeMetodo (tipoVarArg ... nomeArr);

Nel caso in cui in un metodo si vogliano gestire sia parametri normali, sia un parametro vararg, quest'ultimo viene posto per ultimo, in modo tale che vada a raccogliere tutti gli argomenti residuali.
Inoltre, si noti come ogni metodo possa accettare al massimo un solo parametro di tipo vararg.
Esempio:

public class VarArgParam {
	public static void main(String[] args) {
		stampaVoti("Mario", 17, 7, 9, 5, 9, 5, 8);
	}
	public static void stampaVoti(String nome, int età, int ... voti) {
		System.out.print("Ciao " + nome + ", i voti che hai ottenuto sono: ");
		for (int voto : voti) {
		System.out.print(voto + " ");
		}
	}
}
/* Output:
Ciao Mario, i voti che hai ottenuto sono: 7 9 5 9 5 8

Parametri Command Line
...

E' ora possibile capire meglio la definizionemain (String [] args): quello tra parentesi è un parametro del metodo main, un array di tipo String il cui identificatore è args (il nome può dunque essere anche differente, ma args è un identificatore standard per questo utilizzo).
Si ricordi che il metodo main non viene invocato in nessun punto, bensì è l'entry point quando viene eseguito il programma: gli argomenti per tale parametro vengono dunque passati al momento dell'esecuzione del programma da command line, in cui ogni valore separato da spazio diventa il valore di tipo String di una delle posizioni di tale array.
