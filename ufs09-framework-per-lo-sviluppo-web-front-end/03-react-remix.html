03 - React Remix
tags:
  - framework-front-end
  - react
  - js

React è una libreria per la creazione di interfacce utente web e native. Recentemente, Meta (lo sviluppatore di React) ha suggerito l’uso di quattro framework esistenti per React:

  1. Next: Utilizzato principalmente per applicazioni web.
  2. Remix: Altro framework per applicazioni web.
  3. Gatsby: Ottimo per la creazione di siti statici e blog.
  4. Expo: Utilizzato per lo sviluppo di app mobile.

Create React App
...

Per iniziare, useremo il tradizionale framework denominato create-react-app. Questo ci aiuterà a uniformarci agli standard richiesti.

Ottimizzazione tramite import
...

Un trucco utile per ottimizzare il nostro codice è tagliare la funzione del componente Box e crearne un file separato chiamato box.js. In questo modo, possiamo importare il componente Box direttamente nel nostro componente principale (ad esempio, App.js).

Virtual DOM
...

React utilizza un meccanismo chiamato Virtual DOM per confrontare il prossimo DOM con quello attuale ed applicare solo le differenze. Questo rende l’aggiornamento delle interfacce utente più efficiente.

Il formato JSX
...

JSX è una sintassi simile all’HTML, ma in realtà è JavaScript. Grazie a JSX, React è in grado di astrarre la parte di presentazione dal resto del programma.

Passaggio di parametri (props)
...

Possiamo rendere dinamici i componenti passando loro informazioni che ne cambiano aspetto e comportamento. Le props sono un modo per farlo.

Iteratore .map
...

L’iteratore .map ci permette di iterare sugli array e trasformarli in JSX arricchiti con i dati ottenuti. Ad esempio:

const numbers = [1, 2, 3, 4];
const doubledNumbers = numbers.map((number) => number * 2);

// Render the doubled numbers
<ul>
  {doubledNumbers.map((doubledNumber) => (
    <li key={doubledNumber}>{doubledNumber}</li>
  ))}
</ul>

Creiamo un componente JSX
...

Spostiamo i componenti in file dedicati e importiamoli nel nostro componente principale. Ad esempio:

// In Box.js
function Box(props) {
  return <div className="box">{props.children}</div>;
}

// In App.js
import Box from './Box';

function App() {
  return (
    <div>
      <Box>Contenuto del box</Box>
    </div>
  );
}

Modificare gli elementi del DOM
...

Ci sono due modi per cambiare gli elementi del DOM:

  • Imperativo: modifico esplitamente il DOM (con getElementById() e .innerHTML)
  • Dichiarativo: fa utilizzo di stati, la modifica del DOM viene fatta in automatico.

Gestione dello stato: useState
...

Con lo stato, possiamo gestire le informazioni dinamiche all’interno dei nostri componenti. Utilizziamo il hook useState per farlo.

import React, { useState } from 'react';
import './style.css';

export default function App() {
  const [conta, setConta] = useState(5);
  const incrementa = () => {
    setConta((v) => v + 1);
  };
  return (
    <div>
      <h1>Funziona!</h1>
      <p>{conta}</p>
      <button onClick={incrementa}>incrementa</button>
    </div>
  );
}

useEffect dipendenze
...

Il hook useEffect ci permette di scatenare reazioni in base ai cambiamenti di stato. È importante specificare le dipendenze corrette per evitare comportamenti indesiderati.

import { useEffect, useState } from "react"
export const usePosts = () => {
	const [ posts, setPosts ] = useState ([])
	useEffect (() => {
		fetch ('https://jsonplaceholder.typicode.com/posts')
			.then(response => response. json())
			. then (json => {
			setPosts (json)
		})
	},[])
	return { posts }
}

Dati esterni tramite useEffect
...

Iniziamo ad utilizzare un secondo fondamentale hook: useEffect. Questo ci permette di gestire effetti collaterali, come richieste API o aggiornamenti del DOM.

Azione dal figlio al padre
...

Possiamo eseguire codice all’interno dei componenti e comunicare con il componente padre.

Azione dal padre al figlio
...

Invece, possiamo invocare un’azione contenuta nel componente figlio dal componente padre.
