Router
...

  • Scopo: Andare oltre la singola pagina (SPA) creando un sito multi-pagina.
  • Modalità: Importare react-router-dom e definire le “rotte” tramite createBrowserRouter. Sostituire <App /> con il RouterProvider.
  • Cosa succede: Le pagine interne del sito diventano raggiungibili tramite URL. Si può navigare da una pagina all’altra senza ricaricare la app.
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import { createBrowserRouter, RouterProvider } from "react-router-dom";
import Login from "./pages/login";
import Products from "./pages/products";

const router = createBrowserRouter([
  { path: "/", element: <Products /> },
  { path: "/login", element: <Login /> }
]);

const rootElement = document.getElementById("root");
const root = createRoot(rootElement);

root.render(
  <StrictMode>
    <RouterProvider router={router} />
  </StrictMode>
);

Link
...

  • Scopo: Navigare restando all’interno dell’applicazione. Se si usasse <a href="..."> si ricaricherebbe la app ogni volta perdendo stati temporanei.
  • Modalità: Importare Link da react-router-dom e usare Link impostando il parametro “to”.
import { Link } from "react-router-dom";

<Link className="nav-link active" to="/">Home</Link>
<Link className="nav-link" to="/checkout">Checkout</Link>
<Link className="nav-link" to="/login">Login</Link>

Context provider (creazione)
...

  • Scopo: Evitare il passaggio eccessivo di parametri tra componenti (prop-drilling) e accentrare in un posto la logica.
  • Modalità: Creare un hook personale con la logica desiderata, creare un context, e associare il custom hook al valore del provider del context.
export const useCart = () => {
  const [products, setProducts] = useState([]);
  const add = (product) => { setProducts([...products, product]); };
  const remove = (product) => { const filtered = products.filter((p) => p.id !== product.id); setProducts(filtered); };
  return { products, add, remove };
};

export const CartContext = createContext();

export const CartProvider = ({ children }) => {
  const value = useCart();
  return <CartContext.Provider value={value}>{children}</CartContext.Provider>;
};

Context provider (utilizzo)
...

  • Scopo: Utilizzare il context provider.
  • Modalità: Racchiudere il componente finale (RouterProvider in questo caso) con il context provider.
root.render(
  <StrictMode>
    <CartProvider>
      <RouterProvider router={router} />
    </CartProvider>
  </StrictMode>
);

useContext (CartContext)
...

  • Scopo: Usare il custom hook desiderato tramite context nei componenti.
  • Modalità: Creare componenti che usando al loro interno il context. Usare le procedure e gli stati esportati nel custom-hook.
export const CartButtons = ({product = {code:"01"}}) => {
  const { add, remove } = useContext(CartContext);
  const addToCart = () => add(product);
  const removeFromCart = () => remove(product);
  return (
    <div>
      <button onClick={addToCart} className="btn btn-primary">add to cart</button>
      <button onClick={removeFromCart} className="btn btn-warning">remove to cart</button>
    </div>
  );
};

const Cart = () => {
  const { products } = useContext(CartContext);
  return (
    <div>
      <h3>carrello</h3>
      <pre>prodotti: {JSON.stringify(products)}</pre>
    </div>
  );
};

useContext (UserContext)
...

  • Scopo: Gestire l’autenticazione dell’utente e gestire la protezione delle pagine soggette ad autenticazione.
  • Modalità: Creare un context UserContext e relativo UserProvider. Creare un componente RequireAuth che racchiude le pagine da proteggere.
const RequireAuth = ({ children }) => {
  const location = useLocation();
  const auth = useContext(UserContext);

  if (!auth.user) {
    return (
      <Navigate to="/login" state={{ from: location }} replace />
    );
  }

  return children;
};

const Checkout = (props) => (
  <RequireAuth>
    <CheckoutBase {...props} />
  </RequireAuth>
);

useParams (di router)
...

  • Scopo: Gestire pagine dinamiche (es. pagina prodotto, una per ogni prodotto).
  • Modalità: Impostare un path dinamico con “:” e la chiave desiderata. Leggere tramite useParams la chiave dalla pagina (usare la chiave per ottenere il prodotto desiderato).
import PageWrapper from "./page-wrapper";
import { useParams } from "react-router-dom";

const Product = () => {
  const params = useParams();
  return (
    <PageWrapper>
      <h3>product</h3>
      <pre>{JSON.stringify(params, null, 2)}</pre>
    </PageWrapper>
  );
};

export default Product;
